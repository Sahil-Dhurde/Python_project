Algorithm 01

Linear
Regression

A supervised learning algorithm that models the relationship between a dependent variable and one or more independent variables using a straight line. It minimises the sum of squared residuals to find the best-fit line through your data.

📐
How It Works
Fits a hyperplane y = β₀ + β₁x₁ + … + βₙxₙ by minimising Mean Squared Error via Ordinary Least Squares or Gradient Descent.
⚡
Key Assumptions
Linearity, independence, homoscedasticity, normality of residuals, and no multicollinearity among features.
🎯
Evaluation Metrics
MSE, RMSE, MAE, and R² (coefficient of determination) measure prediction accuracy and variance explained.
🔧
Regularisation
Ridge (L2) and Lasso (L1) regression add penalty terms to prevent overfitting and handle high-dimensional data.

Algorithm 02

Random
Forest

An ensemble learning method that builds multiple decision trees during training and outputs the class (classification) or mean prediction (regression) of individual trees. Bagging + random feature selection = powerful generalisation.

🌲
Ensemble Architecture
Combines n bootstrap-sampled decision trees. Each split considers a random subset of √p features, reducing correlation between trees.
🛡️
Bias-Variance Trade-off
Averaging multiple high-variance trees dramatically reduces overall variance without significantly increasing bias — the core benefit of bagging.
📊
Feature Importance
Gini impurity decrease or permutation importance scores rank which features contribute most to predictive power across all trees.
🔬
Out-of-Bag Evaluation
~37% of samples left out of each bootstrap become a free validation set — OOB error is an unbiased generalisation estimate.
Side by Side

Algorithm Comparison

Choosing the right tool depends on your data's structure, interpretability needs, and computational budget.

Criterion Linear Regression Random Forest
Type Parametric Statistical model Non-parametric Ensemble model
Task Regression only Regression & Classification
Interpretability High — coefficients directly explain feature impact Low — black-box; SHAP values needed for explanation
Non-linearity Requires manual feature engineering (polynomial terms) Handles non-linearity and interactions natively
Overfitting Risk Low (unless high-dimensional without regularisation) Low — bagging & random features reduce overfitting
Training Speed Very fast — O(n·p) Slower — O(n·p·k·d) with parallelisation support
Missing Values Sensitive — requires imputation More robust; can handle implicitly
Hyperparameters α (learning rate), λ (regularisation) n_estimators, max_depth, max_features, min_samples
Real-world Applications

Where Each Algorithm Shines

◈ Linear Regression Use Cases
  • House price prediction from size, location, rooms
  • Sales forecasting from advertising spend
  • Risk scoring in insurance & credit
  • Economic modelling — GDP vs. inflation rate
  • Medical dosage estimation based on body weight
  • Energy consumption prediction from temperature
  • Customer lifetime value estimation
◈ Random Forest Use Cases
  • Fraud detection in financial transactions
  • Disease diagnosis from patient features
  • Stock market movement prediction
  • Remote sensing & land cover classification
  • Customer churn prediction
  • Drug discovery & gene expression analysis
  • Anomaly detection in IoT sensor data
